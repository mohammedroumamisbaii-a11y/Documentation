The Metabase Deep Dive

Learn how to deploy, secure, and integrate the world's most popular open-source BI tool into your application.

01. Understanding Metabase

Metabase is an open-source **Business Intelligence (BI)** tool. Think of it as a bridge between your complex database and your human users. It allows you to:

  • Ask Questions: Query your database without writing SQL (Visual Query Builder).
  • Build Dashboards: Combine multiple charts into one live view.
  • Analyze Facebook Data: Perfect for seeing your Airbyte-synced Ads spend and performance.

Figure 1: The Metabase Analytics Interface in action.

02. Deployment (Docker)

In a professional environment, we run Metabase in a **Docker Container**. This ensures that the BI engine has its own isolated memory and won't crash your main app.

Bash Setup 
# Run Metabase on Port 3000
docker run -d -p 3000:3000 --name metabase metabase/metabase

# Access via Browser:
http://localhost:3000

Note: Once running, follow the setup wizard to connect your MySQL database.

03. Professional Embedding (JWT)

You don't want your clients to log into Metabase. You want them to stay in your app. We use Interactive Signed Embedding via JWT (JSON Web Tokens).

How it works:

  1. Metabase gives you an EMBEDDING_SECRET_KEY.
  2. Your Backend signs a message saying: "User 1 is allowed to see Dashboard 5".
  3. The Browser opens an iframe with this signed token.

04. Frontend Rendering

The final result is a seamless, native-looking dashboard within your existing software. No Metabase login required.

Embedded_UI.vue / .html 
<!-- The frontend doesn't know it's Metabase -->
<iframe 
  src="https://your-server.com/embed/dashboard/SIGNED_JWT_TOKEN" 
  width="100%" 
  height="600" 
  frameborder="0" 
  allowtransparency>
</iframe>
