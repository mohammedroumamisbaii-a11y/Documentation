Questions d'Entretien — Ce que les Entreprises Demandent

03

Top Questions Entretien AI Engineer 2025

🔧 RAG Pipeline

Q: "Explique comment tu construirais un RAG de A à Z"
Chunking docs → Embed avec text-embedding-3-small → Store dans pgvector → Query: embed question → cosine similarity search → top-k chunks → prompt LLM avec contexte → réponse
Q: "Comment tu gères les documents longs ?"
Chunking strategy : semantic chunking > fixed-size. Overlap entre chunks (20%). Metadata filtering pour réduire le search space.

🤖 Agents & Tools

Q: "Différence entre Chain et Agent ?"
Chain = séquence fixe d'étapes. Agent = LLM décide dynamiquement quelle action prendre selon le contexte. Agent est flexible, Chain est prévisible.
Q: "Quand tu utilises pas un agent ?"
Quand le workflow est simple et prévisible — les agents ajoutent de la latence et des coûts. KISS principle.

📊 Évaluation

Q: "Comment tu évalues ton système RAG ?"
RAGAS : faithfulness (réponse basée sur contexte ?), answer relevancy (répond à la question ?), context precision. + Tests manuels avec ground truth dataset.
Q: "Comment réduire les hallucinations ?"
RAG avec grounding, temperature basse (0.1-0.3), demander au modèle de citer ses sources, fallback "je sais pas" si confidence faible.

⚡ Performance & Coûts

Q: "Comment tu optimises les coûts LLM ?"
Caching des réponses fréquentes, prompt compression, choisir le bon modèle (haiku pour tâches simples, opus pour complexe), batch processing.
Q: "Latence trop haute, que faire ?"
Streaming responses, réduire context window, async calls parallèles, semantic caching avec Redis.

🏗 Architecture

Q: "RAG vs Fine-tuning, quand choisir ?"
RAG : données fréquemment mises à jour, besoin de citer les sources, budget limité. Fine-tuning : style/ton très spécifique, tâche répétitive très précise, latence critique.
Q: "Comment tu versiones tes prompts ?"
Git pour les prompts, A/B testing avec métriques, PromptLayer ou LangSmith pour tracking.

🔒 Production & Sécurité

Q: "Prompt injection, comment tu te protèges ?"
Input sanitization, instructions claires dans system prompt, validation output, LLM-as-judge pour détecter les réponses hors contexte.
Q: "Comment tu monitores en production ?"
Log tous les appels LLM (input/output/latency/cost), alertes sur taux d'erreur, LangSmith ou Langfuse pour observabilité.
