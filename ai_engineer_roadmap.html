Embedding Analytics into Modern Applications

A comprehensive guide for developers to install, secure, and integrate professional Business Intelligence dashboards into any software ecosystem.

01. Server-Side Deployment

To run a BI engine reliably, Docker is the industry standard. It ensures that the environment is consistent from development to production.

A. Local/Production Launch

Run the following command to pull the latest image and map the persistence port (3000):

docker run -d -p 3000:3000 --name bi_engine metabase/metabase
💡 Production Tip: For high-availability, always map a persistent volume to /metabase.db or connect Metabase to an external Postgres database to store its configuration metadata.

02. Data Source Orchestration

Once the server is live at http://your-server-ip:3000, you must bridge it with your application's data.

  • Compatibility: Connects to MySQL, PostgreSQL, MongoDB, SQL Server, and BigQuery.
  • Read-Only Access: Always create a specific database user with SELECT only permissions for the BI engine. This prevents accidental data deletion via the BI interface.
  • Sync Frequency: Configure the engine to scan for schema changes every hour to stay updated with your app's migrations.

03. Implementing Signed Embedding

Standard sharing is public. In professional apps, we use Signed Embedding to ensure that only authenticated users can see specific charts.

The Authentication Logic:

The application generates a JSON Web Token (JWT) signed with a secret key provided by the BI engine. This token tells the engine: "I am a trusted app, and I allow this user to see Dashboard X."

// Logic Flow (Backend) 1. Define the Dashboard ID. 2. Set an expiration time (e.g., +15 minutes). 3. Create a payload: { "resource": {"dashboard": 123}, "params": {} } 4. Sign the payload using HS256 algorithm with your SECRET_KEY.

04. Application Frontend Integration

The final step is rendering the secured dashboard inside your application's UI. This is done via an <iframe>.

Implementation Example:

<!-- The frontend receives the Signed URL from the backend --> <iframe src="https://stats.yourapp.com/embed/dashboard/SIGNATURE_HERE#bordered=true&titled=false" frameborder="0" width="100%" height="800" allowtransparency ></iframe>
🔒 Multi-tenancy: If your app has multiple clients, you can pass locked parameters in the JWT. This ensures Client A can never see Client B's data, even if they change the ID in the URL.
